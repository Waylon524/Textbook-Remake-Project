# Open-Knowledge-Base 开源知识图谱计划

## 项目简介
本项目致力于利用人工智能（AI）工作流与现代文本处理技术，将客观存在的科学基础知识（如数学分析、生物化学等领域的客观规律、定理与公式）进行结构化、原子化重构。通过提炼公有领域文献、开源教参及客观事实，利用 AI 重新撰写与逻辑编排，生成一套逻辑更清晰、无版权争议、更适合现代学习习惯的全新开源电子知识库。
## 核心愿景
• 知识碎片化： 将庞大的学科体系拆解为原子化的独立知识点文件。    
• 原创性重构： 剥离传统教材的既有文字表达，仅提取核心客观事实（Ideas），利用 AI 生成更符合现代认知直觉的全新原创解析（Expressions）。    
• 数字化进化： 从传统的线性阅读转向结构化的知识网络，打造互联互通的数字笔记体系。    
## 技术栈
• 数据采集引擎： PaddleOCR（负责公有领域公式与图表的无损提取）。  
• 辅助生成与校对： VS Code + GitHub Copilot + LLM 提示词工程。  
• 处理脚本： Python（自动化批处理、知识点路由映射）。  
• 存储格式： Markdown (LaTeX 渲染科学公式)。  
## 操作流程 (Workflow)
### 第一阶段：原始数据数字化  
1. 收集公有领域文献（Public Domain）、**遵循 CC 协议的开源教参（如 OpenStax）**以及客观存在的科学公式库。
2. 使用工具提取上述合规材料中的复杂数学公式与客观图表结构。
3. 初步输出为未经组织的纯事实型 Markdown 节点。
### 第二阶段：AI 辅助清洗与修正  
1.	在 VS Code 中打开原始 Markdown 文件。
2.	利用 GitHub Copilot 或 AI 脚本对 OCR 识别错误（特别是复杂的符号和算式）进行语法修正。
3.	确保所有数学公式符合标准的 LaTeX 语法。
### 第三阶段：知识点拆分  
1.	按照“学科 -> 大章节 -> 具体知识点”的逻辑层级进行拆解。
2.	每一个独立的知识点生成一个独立的 .md 文件。
3.	剔除所有来源文献的特有排版、例题和叙述性文字，仅保留客观事实本身。
### 第四阶段：多源整合与优化  
1. 向 AI 模型输入剥离了表达形式的纯知识点事实。
2. 通过精心设计的 Prompt，要求 AI 完全使用全新的语言结构和现代教学逻辑，编写出兼具直观性与严谨性的知识点解析。
3. 由 AI 生成全新的辅助理解例题（绝不照搬任何现代商业教材题库）。
### 第五阶段：人工审核  
1.	关键节点审核：人工检查 AI 整合后的内容是否存在逻辑漏洞或事实错误。
2.	确保数学推导的准确性以及专业术语的统一。
### 第六阶段：发布与展示  
1.	GitHub 阶段：所有重制后的知识点以 Markdown 仓库形式存储，方便版本管理。
2.	网站阶段：后期将内容迁移至自建 Web 端（如基于 React 的在线文档系统），提供更佳的阅读与检索体验。
